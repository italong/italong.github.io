2.面向对象的特点

答：首先面向对象是指，将一个个事务抽象成为一个个行为，这样做的目的不是为了去关注它一个个的行为，而是为了避免不必要的公共部分，提高代码的复用性；面向对象有三大特性：封装、继承、多态；

封装是指将一个对象的属性私有化，同时提供给能给外界访问的get/set方法；继承则是说子类继承父类，它拥有父类的所有属性和方法，但是无法访问父类的私有属性和方法，它可以用自己的方式实现父类的方法，同时它也拥有自己的属性和方法；多态是指引用变量在编程时，无法知道它引用的数据类型，和它引用的到底是哪个方法；只有在运行时才能知道，也就是说该引用变量到底指向哪个类的实例对象，该引用变量发出的方法到底调用哪个类实现的方法；多态的实现主要分为两种，重载和重写，重载呢是说编译时多态，重写呢则是运行时多态；重载是说和（）拥有相同的返回类型，但是它可能参数类型不同，参数个数不同或者两者皆不同；重写则是说，和其方法返回类型不同，同时不能抛出更多异常，要比原方法更高效；

3.抽象与接口的区别

答：第一，接口的方法默认是public，同时方法没有具体实现，但是从1.8开始，方法有默认实现；而抽象类可以有非抽象方法；第二，接口中变量除了static （final）之外，不能有其他变量，而抽象类中则不一定；第三，接口的默认修饰符是public，而抽象类的则是dafult public protect，而它不能使用private是因为它本身就是为了被重写才定义的；（第四，一个类可以实现多个接口，但只能实现一个抽象类，接口本身可以通过extends关键字扩展多个多个接口）；第五，抽象是对类的抽象，是一种模板设计；接口是对行为的抽象，是一种行为规范；

4.==和equals的区别

答：== 在当比较的的基本数据类型时，比较的是值，在比较的是引用数据类型时，比较的是地址；equals在默认情况下比较两边的对象，即比较内存地址，而自定义后，则是类覆盖了默认的equals方法，比较的是两边的值；

字符型有char boolean

答：装箱是说将基本数据类型装成引用数据类型；而拆箱则相反；

7.String StringBuilder StringBUffer的区别

8.什么是序列化

答：就是将java对象装成二进制流；而反序列化就是将二进制流拆成java对象；类比生活中装箱拆箱；三大序列化手段

9.final finally finalize的区别

答：final修饰的一般是类 方法 变量，类不可被继承，方法不可被重写，变量不可变用；

finally一般用来作捕获异常处理的必要流程trycatchfinally一般是在需要释放资源时调用，system.exit(0)可以阻止其启动；finalize是object包下的类，也就是说每个类都拥有，这个方法在GC启动，

4.hashmap的底层原理

首先是散列表：红黑树可以避免DDOS（daosi）攻击

答：首先java8中hashMap采用数组+链表/红黑树的数据结构；同时在查询过程中，计算hash值 相同则替换，不同则存入链表或红黑树；获取数据通过key的hash值来计算得到数组下标从而获取元素；

5.hashmap的put方法的具体流程

答：1.判断数组长度是否为空null，为空则调用resize方法扩容或初始化；

2.计算hash值获取数组下标

3.判断当前table[i]==null，为空则直接插入；

4.不为空则需要先判断该位置是否为红黑树,如果是则走红黑树的put流程；若不是，则遍历链表，在链表中若存在key相同的值，则直接覆盖，若不存在，则需要在链尾判断next是否为空，为空则添加，会判断当前链表长度是否大于8，若大于则需要转化为红黑树，走红黑树的put流程；

5.最后判断实际的键值对数量是否大于最大容量也就是数组长度*阈值，大于则需要走之前的resize扩容；

6.hashmap的扩容流程

答：1.在第一次添加元素时，进行初始化容量一般为16；之后每次判断是否需要扩容的条件则是使用长度是否大于扩容阈值；

2.初始化后扩容一般是扩容2倍；

3.扩容之后先新建数组，遍历老数组，先判断hash值模与newMap-1是否为0，若是则直接添加，不是则判断是否为红黑树，若是则走红黑树添加；若不是则遍历链表，可能需要拆分链表，判断hash值&oldMap是否为0，若是则newMap[j] = oldMap[j]，若不成立，则赋给newMap[j+oldMap] ；

1.索引

1.什么是索引？

答：知道一些的，索引是帮助mysql提高查询数据效率的数据结构，它可以提高数据查询效率，降低数据库IO成本；同时，可以提高数据排序速度，帮助数据库排序，从而减少cpu的压力；

2.索引的数据结构？

答：索引底层采用的B+树，它相对于B树来说，B树的非叶子节点和子节点都会存储数据，而B+树则只有叶子节点存储数据；同时，它阶数更短，路径更低；在叶子节点上采取双向链表的方式，可以快速扫库和区间查询；

2.存储引擎

MYSQL支持的存储引擎？

答：InnoDB存储引擎是mysql5.5以后默认的存储引擎，它支持事务、外键、行级锁和表级锁；

MyISAM是早期的存储引擎，不支持事务，只有表级锁，也没有外键，用的不多；

Memory主要把数据存储在内存，支持表级锁，没有外键和事务，应用也不多；

存储引擎在mysql体系结构哪一层，主要特点？

答：MYSQL主要分为连接层服务层引擎层存储层；连接层主要存放JDBC ODBC连接工具；服务层则是一些缓存SQL解析器、连接池等；引擎层存放存储引擎；存储层则是一些系统文件和其他文件、日志；

InnoDB是一种兼具高可靠性和高性能的存储引擎；从5.5开始mysql默认的存储引擎，它支持事务（ACID）外键（保证数据的完整性和可靠性）行级锁（支持并发访问性能）表级锁；ibd后缀文件(表空间文件) frm文件后缀（表结构文件）

什么是聚簇索引和非聚簇索引？回表？

答：聚簇索引是指将主键和数据都存放在叶子节点，其存储的是整行数据；而非聚簇索引是指将主键和数据分开存储，一般我们自己定义的索引就是非聚簇索引；回表是指通过二级索引得到对应的主键值，再到聚集索引中查询得到整行数据，这个过程就是回表查询；

什么是覆盖查询？mysql超大分页怎么处理？

答：覆盖查询是指使用了索引在返回的列中全部都能查询得到，比如说我们查询条件人名，我们以人名创建了索引，那么我们返回的列中必须只有人名和id；mysql超大分页主要指的是通过limit进行分页处理时，最开始数据量可能不大，当达到百万级时，通过limit查询会减少它的查询效率，因为它每次查询得到的结果都需要将前面的都查询出来，然后丢弃，一般我们采取覆盖查询+子查询的方式优化，通过覆盖查询，先查询出id，再通过id来获取得到该字段的数据

索引创建原则？

答：一般创建索引式需要在数据量很大的情况下去创建的，像是数据量达到了10w条以上，或者说该字段查询很频繁的情况我们就需要创建索引；尽量选择区分度高的创建唯一索引，这样使用索引的效率就越高，如果式字符串类型，字段的长度较长，我们就可以创建前缀索引；当然并不是所有都应该创建索引，我们也要控制索引的数量，因为增加索引也会导致新增加的速度变慢；

什么时候索引会失效？

答：失效的场景有很多，我说一些我具体遇到的失效场景，像是返回的列没有能在覆盖索引中全部查出，也就是跳过了索引的列没有满足最左匹配原则，又或者范围查询右边的列，在索引列上进行一系列运算操作；字符串不加单引号（查询优化器会优化将其类型转化） ，模糊查询以%开投也会造成索引失效；一般我们通过explain来查看具体的效果和优化方案；

2.事务

事务是什么？

答：事务时一组操作的集合，它会将所有的操作一起向系统发送提交或撤销的请求，也就是说要么都成功要么都失败；

事务的特性详细说一下？

答：事务具有四大特性：原子 一致 隔离 持久 ACID：我举一个例子具体说明一下：A向B转账500，原子性体现在转账提交成功均一起提交；一致性体现在A转500 B收500，总额一致；隔离性则是转账过程中不受其他事务影响；持久性则是说最后完成后数据持久化也就是落盘；

undo log 和redo log 的区别？

答：redo log 记录的是数据页的物理变化，当数据库宕机后可以同步数据一般是通过redo log buffer 和 redolog file来操作，当它大量的更改删除操作时，我们先在缓冲池中进行操作，存到redo log buffer中再同步到磁盘的redo log file文件中，这时候如果宕机了，我们之后只需要通过redo log file 来恢复数据就可以了，这里并不是进行一个多余的操作，是为了将之前杂乱的IO能够有序化处理，保证了事务的原子性；而undo log则是逻辑日志，用于记录数据被修改前的数据，也就是说如果有一个delete就在undo log中有一个insert语句，用来进行回滚操作，一般用于MVCC中，保证了事务的一致性和持久性；

事务中的隔离性如何保证的呢？

答：一般通过加锁（排他锁）和MVCC多版本并发控制来进行具体操作，排他锁指的是一个事务对该行加排他锁之后，其他事务不能获取该行的其他锁；

4.乐观锁和悲观锁

答：悲观锁：

悲观锁认为被它保护的数据是极其不安全的，每时每刻都有可能被改动，⼀个事务拿到悲观锁后，其他任何事务都不能对该数据进⾏修改，只能等待锁被释放才可以执⾏。数据库中的⾏锁，表锁，读锁，写锁均为悲观锁。

乐观锁：乐观锁认为数据的变动不会太频繁。乐观锁通常是通过在表中增加⼀个版本(version)或时间戳(timestamp)来实现，其中，版本最为常⽤。事务在从数据库中取数据时，会将该数据的版本也取出来(v1)，当事务对数据变动完毕想要将其更新到表中时，会将之前取出的版本 v1 与数据中最新的版本 v2 相对⽐，如果 v1=v2，那么说明在数据变动

期间，没有其他事务对数据进⾏修改，此时，就允许事务对表中的数据进⾏修改，并且修改时 version会加 1，以此来表明数据已被变动。如果，v1 不等于 v2，那么说明数据变动期间，数据被其他事务改动了，此时不允许数据更新到表中，⼀般的处理办法是通知⽤户让其重新操作。不同于悲观锁，乐观锁通常是由开发者实现的。
